格式要求

　　为了让教程条理清晰，易于阅读，VIYF 对文档排版有下列要求：

1. 采用 Markdown 格式进行撰写

　　Markdown 是一种纯文本格式，可以用它进行日常的文档排版工作。我们会在下文介绍 Markdown 的语法。

2. 合理分配板块（标题）

　　一篇文档是由不同的板块组成的，而每个板块都应该有它的标题。通常，标题有六级，在撰写时，文档标题应该采用一级标题，文档中的板块采用二级标题，板块中的子版块用三级标题，以此类推，但请注意不要把标题的层次结构搞错了。

　　没有一成不变的方法来确定何时该分配板块，原则是，介绍完一个项目之后就拆分板块。例如，下载读屏软件，分为读屏软件简介，官方网站地址，下载方法和安装方法这几个步骤，这几个步骤都可以分配板块。

3. 正确创建段落

　　在 Markdown 格式中，要创建段落，是在当前段落到下一段之间保留一个或多个空行。如果要强制换行，请在当前行末尾键入两个或多个半角空格。

　　每一段的开头应该保留两个全角空格，要键入全角空格，请按下 Shift + 空格键切换到全角输入模式，输入完后再按 Shift + 空格切换到半角输入模式。如果使用微软拼音输入法，请在“设置”的“时间和语言”中，打开“语言”，打开“中文(简体，中国)”然后点击“选项”按钮。在这里，点击“微软拼音 输入法”，再次点击“选项”按钮。切换到“按键”，打开。找到“Shift + 空格” 单选按钮，按空格键选定，然后退出设置程序。

4. 突出重点

　　在一篇文章中，肯定会有某句话，或者某个关键词需要特别注意。对于这些句子或者词语，应该加粗以便让读者注意。

5. 列出本篇文章的主要内容

　　把文章写好后，最好在开头加上一个板块，叫做“通过这节的学习，同学们将会掌握：”，里面以有序列表的形式列出本篇文章的主要内容。

6. 防止代码被当成 Markdown 格式的一部分

　　在文档中，要加入代码，无论是命令行，还是 cmdlet，都应该将它写到代码块当中。


Markdown 的语法

　　这是撰写教程时不可不知的 Markdown 语法，请记住它！

标题

　　以半角警号开头的行是标题，警号后面跟着一个空格，警号的数量表示标题的层级，比如：

# 一级标题

## 二级标题

### 三级标题

###### 六级标题

粗体和斜体

　　用半角星号括起来的文字可以实现倾斜和加粗的效果。左右各一个星号是倾斜，两个星号是加粗，三个星号是加粗倾斜。例如：

*倾斜*

**加粗**

***加粗倾斜***

列表

　　以点空格开头的行是无序列表，以数字加点空格开头的行是有序列表，行与行之间不能有空行。例如：

这是无序列表

日本的大城市有：

. 东京
. 神户
. 名古屋
. 札幌
. 京都

这是有序列表

通过本节的学习，同学们将会掌握

1. 什么是硬件，什么是软件
2. 微软操作系统是怎样发展起来的
3. 对于视障同学而言，Windows 10 系统有哪些令人兴奋的功能
4. 如何启动 Windows 10 的讲述人
5. 熟悉讲述人的快捷键

插入链接

　　用半角方括号把链接的名称包含在其中，紧随其后用半角小括号把链接的网址包括在其中。例如：

[必应](https://cn.bing.com/)

插入图片

方法类似插入链接，唯一不同是要在左方括号的左边加上半角感叹号，其中的信息用作图片的说明，以便于让视障同学大概了解图片的内容。例如：

![极乐世界](https://www.example.com/bliss.jpg)

插入代码块

　　使用半角反撇号将命令行及 cmdlet 包括在其中，如果需要一次展示多行代码，请在代码的上一行及代码的下一行分别使用三个反撇号。例如：

一行

`ping www.viyf.org`

多行

```
nslookup www.viyf.org
ping www.viyf.org
```

常用的 Markdown 代码就说到这里，如果有兴趣，可以到 https://docs.github.com/cn/free-pro-team@latest/github/writing-on-github/basic-writing-and-formatting-syntax 了解更多。
